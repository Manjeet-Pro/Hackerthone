HTML Day03: File Path System

Lecture 1 Lecture 2 Lecture 3 Lecture 4 Lecture 5 Lecture 6

Step 1: The First Principle (The Fundamental Truth)

A website is not a single file. It consists of multiple files such as HTML, CSS, JavaScript, images, videos, and fonts.

These files are organized into folders, similar to how documents are organized on your computer.

Step 2: The Core Problem

If your index.html needs to display an image logo.png, how does the HTML file tell the browser where to find it?

You cannot just write src="logo.png" because the image might be inside a folder or somewhere else.

The browser needs a precise file path (address) to locate the file correctly.

A) Relative File Paths (Most Important)

Relative file paths are specified relative to the current file location.

my-website/
├── index.html
├── about.html
│
├── images/
│   ├── logo.png
│   └── hero.jpg
│
└── pages/
    ├── contact.html
    └── terms.html

Scenario 1: Linking to a file in the SAME folder

You are in index.html and want to link to about.html. They are in the same folder.


<a href="about.html">About Us</a>

Scenario 2: Linking to a file in a SUB-FOLDER

You want to display logo.png inside the images folder from index.html.


<img src="images/logo.png" alt="My Website Logo">

Scenario 3: Linking to a file in a PARENT FOLDER

You are in contact.html inside pages/, want to display logo.png inside the images/.


<img src="../images/logo.png" alt="My Website Logo">

../ means "go up one folder".

B) Absolute File Paths

Absolute paths specify the full URL to a resource on the web.

Used only for linking external files (e.g., fonts, images hosted elsewhere).

<a href="https://www.google.com">Search on Google</a>

<img src="https://example.com/images/logo.png" alt="Logo">

Why Not Use Absolute Paths for Your Own Files?

Absolute paths like C:/Users/Arjun/Desktop/my-website/images/logo.png only work on your computer and break on web hosting servers.

Browsers restrict direct access to local files for security reasons.

Use relative paths so your website works on any computer or server.

Summary of Key Path Rules

  • Use relative paths for your own website files (images, CSS, other pages).
  • Use absolute paths only for external resources.
  • filename.html - same folder
  • folder/filename.html - subfolder
  • ../filename.html - parent folder
  • https://example.com/filename.html - external absolute URL

HTML Boilerplate Code Example

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Website</title>
</head>
<body>

  <!-- Your content goes here -->

</body>
</html>

This is the required HTML setup to ensure proper rendering and compatibility.
